- Finalize repository structure and coding conventions
- Implement reusable data loaders and validation rules

- Add MIP-based sequencing for constrained resources
- Include setup/changeover and lateness impacts
- Evaluate sequence quality and stability trade-offs

- Introduce disruption-aware re-planning logic
- Balance urgent service demand with schedule stability
- Add operational KPIs for re-plan quality and execution risk
